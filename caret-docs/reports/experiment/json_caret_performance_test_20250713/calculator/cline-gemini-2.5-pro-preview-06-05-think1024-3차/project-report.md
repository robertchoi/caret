# 웹 계산기 프로젝트 보고서

## 1. 프로젝트 개요
이 프로젝트는 간단한 웹 기반 계산기 애플리케이션을 개발하는 것을 목표로 합니다. 사용자는 기본적인 사칙연산, 소수점 계산, 백분율, 부호 변경 등의 기능을 사용할 수 있습니다. 개발 과정에는 요구사항 명세서 작성, 테스트 주도 개발(TDD) 방식의 코드 구현, 그리고 최종 보고서 작성이 포함됩니다.

## 2. 구현된 기능 목록
- **기본 사칙연산:** 덧셈, 뺄셈, 곱셈, 나눗셈
- **숫자 및 소수점 입력:** 0-9 숫자와 소수점(.) 입력 가능
- **초기화 (AC):** 모든 입력을 지우고 계산기를 초기 상태로 되돌립니다.
- **부호 변경 (+/-):** 현재 숫자의 부호를 전환합니다.
- **백분율 (%):** 현재 숫자를 100으로 나눈 값으로 변환합니다.
- **오류 처리:** 0으로 나눌 경우 "Error" 메시지를 표시합니다.

## 3. 사용된 기술 스택
- **HTML:** 애플리케이션의 기본 구조를 정의합니다.
- **CSS:** 애플리케이션의 스타일과 레이아웃을 담당합니다.
- **JavaScript:** 계산기 로직과 사용자 인터랙션을 처리합니다.
- **Test Framework:** 순수 JavaScript로 작성된 간단한 테스트 프레임워크를 사용하여 핵심 로직을 검증합니다.

## 4. 프로젝트 구조
```
/calculator/cline-gemini-2.5-pro-preview-06-05-think1024-3차/
├── calculator-spec.md  # 요구사항 명세서
├── index.html          # 메인 HTML 파일
├── style.css           # 스타일시트
├── script.js           # 계산기 로직
└── script.test.js      # 핵심 로직 테스트 코드
```

## 5. TDD 접근 방식 요약
본 프로젝트는 테스트 주도 개발(TDD) 방법론에 따라 진행되었습니다.

1.  **요구사항 분석 및 테스트 케이스 정의:** `calculator-spec.md` 파일에 명시된 핵심 로직(덧셈, 뺄셈, 0으로 나누기 등)을 기반으로 테스트 케이스를 먼저 정의했습니다.
2.  **테스트 코드 작성:** `script.test.js` 파일에 순수 JavaScript로 간단한 테스트 프레임워크를 만들고, 정의된 테스트 케이스들을 코드로 작성했습니다. 이 시점에서는 모든 테스트가 실패하는 상태입니다.
3.  **실제 코드 구현:** `script.js` 파일에 테스트를 통과할 수 있는 최소한의 기능 코드를 작성했습니다. `calculate`, `resetCalculator`, `toggleSign`, `getPercentage` 등의 함수를 구현하여 모든 테스트 케이스를 통과시켰습니다.
4.  **리팩토링 및 UI 구현:** 테스트가 통과된 후, 코드의 가독성을 높이고 구조를 개선하는 리팩토링을 진행했습니다. 마지막으로 `index.html`과 `style.css`를 작성하여 사용자 인터페이스를 완성했습니다.

이러한 TDD 접근 방식을 통해 코드의 안정성과 신뢰성을 높이고, 요구사항이 정확하게 반영되었는지 지속적으로 검증하며 개발을 진행할 수 있었습니다.
