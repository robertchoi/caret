# 🤖 AI 작업 인덱스 가이드

**목적**: AI가 작업 성격을 파악하고 필요한 문서만 선택적으로 읽어 컨텍스트 효율성을 극대화

## 📋 **단계별 작업 프로세스**

### Phase 0: 필수 선행 독해 (항상 읽기)

AI는 모든 작업 시작 전에 다음 문서들을 **반드시** 읽어야 합니다:

1. **`.caretrules`** - 프로젝트 절대 규칙
2. **`caret-docs/development/index.mdx`** - 개발 가이드 개요 (빠른 스캔)
3. **이 파일 (`ai-work-index.mdx`)** - 작업 인덱스 가이드

### Phase 1: 작업 성격 판단 (키워드 기반)

사용자 요청에서 키워드를 추출하여 작업 성격을 분류:

## ⚡ **효율적인 독해 전략**

### 🎯 **단계별 접근**

1. **작업 성격 파악** (30초): 키워드 추출 → 카테고리 분류
2. **필수 문서 선택** (1분): 매핑 테이블 기반 문서 선택
3. **선택적 독해** (5-10분): 필요한 섹션만 집중 독해
4. **실행 전 체크** (1분): .caretrules 체크리스트 재확인

### 📚 **독해 우선순위**

1. **High Priority**: 직접 관련된 핵심 문서 (1-2개)
2. **Medium Priority**: 간접 관련 문서 (필요시만)
3. **Low Priority**: 참고용 문서 (링크만 확인)

### 🔍 **스마트 독해 방법**

- **목차 우선 스캔**: 관련 섹션만 식별
- **키워드 검색**: 구체적인 내용 빠르게 찾기
- **예제 코드 우선**: 실제 구현 패턴 확인
- **체크리스트 활용**: 놓치기 쉬운 필수 사항 점검
