# 작업 로그 작성 및 사용 가이드 (For AI & 개발자)

## 1. 개요

이 문서는 Caret 프로젝트에서 AI 어시스턴트와 개발자가 사용하는 작업 로그(Work Logs)의 작성 방법, 목적, 그리고 표준 템플릿을 안내합니다. 작업 로그는 일일 작업 진행 상황, 주요 결정 사항, 발생한 이슈, 그리고 다음 작업 계획 등을 기록하여 투명하고 효율적인 협업을 지원합니다.

**참고**: 일반적인 문서 작성 표준, 스타일, 용어 사용, 파일 명명 규칙 등 프로젝트 전반의 문서화 지침은 [Project Documentation Guide](../../development/documentation-guide.md)를 참조하세요.

작업 로그는 각 사용자별로 `docs/work-logs/{username}/` 디렉토리에 `YYYY-MM-DD.md` 형식으로 매일 생성됩니다.

## 2. 작업 로그의 목적

*   **진행 상황 추적**: 당일 수행한 작업과 달성한 목표를 명확히 기록합니다.
*   **이슈 관리**: 발생한 문제점, 해결 노력, 미해결 이슈 등을 기록하여 공유합니다.
*   **맥락 공유**: 다음 작업 세션 또는 다른 팀원이 작업을 이어받을 때 필요한 맥락 정보를 제공합니다.
*   **커뮤니케이션 보조**: AI와 개발자 간의 비동기적 커뮤니케이션을 지원합니다.

## 3. 작업 로그 업데이트 시점 (AI 기준)

AI는 다음 시점에 반드시 작업 로그를 업데이트해야 합니다:

1.  **작업 세션 시작 시**:
    *   로그 파일 생성 (해당일 첫 작업 시) 또는 기존 파일에 추가.
    *   날짜, 세션 시작 시간 명시.
    *   오늘 진행할 주 작업 목표 (Task 번호, 핵심 내용) 간략히 언급.
    *   (예: "[2024-03-20 09:00] 작업 시작. Task #001 - 웹뷰 UI 에러 해결 및 로깅 시스템 검증 진행 예정.")

2.  **주요 체크리스트 항목 완료/진행 시**:
    *   작업 체크리스트(`...-02-action-checklist-*.md`)의 주요 항목을 완료했거나, 상당 부분 진척이 있었을 때 기록합니다.
    *   완료된 항목, 간략한 결과 또는 다음 단계 언급.
    *   (예: "[11:30] Task #001 - 체크리스트 1.1 `WelcomeView.tsx` 오버레이 문제 해결 시도 중, Vite 설정 분석 완료. alias 관련 코드 수정 진행.")

3.  **중요한 이슈 발생 및 해결 시도/결과 기록 시**:
    *   예상치 못한 에러, 기술적 난관, 결정이 필요한 사항 등이 발생했을 때 기록합니다.
    *   문제 상황, 시도한 해결 방법, 결과 (성공/실패/부분 성공), 개발자에게 질문할 내용 등을 간략히 포함합니다.
    *   (예: "[14:00] 이슈: `WelcomeView.tsx` 에서 여전히 `cannot find module '@cline/utils/...'` 에러 발생. `tsconfig.json` 경로 별칭 재확인 및 `vite.config.ts` 설정 검토 필요. 개발자께 관련 설정 조언 요청 예정.")

4.  **작업 세션 종료 시**:
    *   세션 종료 시간 명시.
    *   금일 완료한 주요 작업 항목 요약.
    *   미완료된 작업 및 현재 상태.
    *   해결되지 않은 주요 이슈 요약.
    *   다음 작업 세션에서 진행할 예정인 작업 간략히 언급.
    *   (예: "[18:00] 작업 종료. 금일 `WelcomeView.tsx` 렌더링 에러 분석 및 Vite 설정 일부 수정. 로깅 시스템 기본 검증 완료. 미해결 이슈: `@cline/utils` 경로 문제. 내일 오전 해당 이슈 집중 해결 예정.")

## 4. 작업 로그 작성 형식 및 내용

*   **간결성**: 핵심 내용 위주로 간결하고 명확하게 작성합니다.
*   **사실 기반**: 추측보다는 확인된 사실을 바탕으로 기록합니다.
*   **시간 기록**: 주요 이벤트 발생 시 또는 로그 항목 작성 시 시간을 함께 기록합니다 (예: `[HH:MM]`).
*   **작업 번호 참조**: 관련된 작업이 있을 경우 Task 번호를 명시합니다 (예: `Task #001`).
*   **Markdown 활용**: 가독성을 높이기 위해 Markdown의 목록, 강조 등을 적절히 사용합니다.

### 4.1. 권장 템플릿 (일일 로그 파일 내용)

```markdown
# YYYY-MM-DD 작업 로그 - AI

## 작업 목표 (오늘)
*   Task #00X: [주요 목표 1]
*   Task #00Y: [주요 목표 2]

## 타임라인

[09:00] 작업 시작. Task #00X - [세부 목표] 진행 예정.

[10:30] Task #00X - 체크리스트 항목 [X.Y] 완료. [간단한 결과 또는 다음 단계].

[11:15] 이슈 발생: Task #00X 진행 중 [문제 상황 요약]. [시도한 해결 방법] -> [결과].

[14:00] 개발자와 논의: [논의 주제] -> [결정 사항 또는 다음 액션].

[16:45] Task #00X - 체크리스트 항목 [X.Z] 관련 [진행 상황 기술].

[18:00] 작업 종료.
*   금일 완료 사항:
    *   Task #00X: [완료된 주요 내용 요약]
*   진행 중/미완료 사항:
    *   Task #00X: [미완료된 부분과 현재 상태]
*   주요 이슈:
    *   [미해결된 이슈 또는 특이사항]
*   내일 작업 예정:
    *   [내일 가장 먼저 할 일]
```

## 5. 작업 로그 활용 (개발자 기준)

*   AI의 일일 작업 진행 상황을 파악합니다.
*   AI가 직면한 문제점이나 필요한 지원 사항을 확인합니다.
*   비동기적 피드백이나 다음 작업 지시를 위한 기초 자료로 활용합니다.

이 가이드라인은 효과적인 작업 추적과 명확한 커뮤니케이션을 위해 중요합니다. AI는 항상 이 지침을 따르도록 노력해야 합니다.
